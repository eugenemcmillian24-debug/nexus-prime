# 方法论速查表

每个速查表对应工作流中的一个关键步骤。这些不是模板——理解背后的原则，然后根据具体问题灵活运用。

---

## 问题定义检查清单

对应 Step 1。一个好的问题定义应该能让任何人读完后知道"我们在解决什么"和"我们不解决什么"。

**核心要素：**

- **What** — 核心问题是什么？用一句话描述，要具体到能指导拆解方向。  
  ❌ "分析新能源行业"（太宽泛，无法指导后续）  
  ✅ "评估 A 公司在中国新能源乘用车市场的竞争地位，并识别增长机会"

- **Why** — 为什么这个问题现在重要？背后的业务驱动力是什么？  
  理解紧迫性有助于判断分析的深度和速度。

- **Scope** — 边界在哪里？  
  明确包含和不包含的内容，防止分析无限发散。地理范围？时间跨度？业务线？

- **Deliverable** — 期望什么形式的输出？  
  PPT（几页？给谁看？）、Markdown 报告、Excel 模型、口头汇报框架？

- **Success Criteria** — 怎样算"做好了"？  
  有可操作的建议？量化了机会规模？说服了决策者？

**常见陷阱：**
- 问题太大，一次分析不完 → 缩小范围或拆成多个子项目
- 问题太小，不值得结构化分析 → 直接给建议，不需要走全流程
- 问题已经暗含了答案 → 退一步，先理解真正的问题是什么

## MECE 拆解技巧

对应 Step 2。MECE（Mutually Exclusive, Collectively Exhaustive）是 Issue Tree 的质量标准。

**MECE 原则：**
- **Mutually Exclusive（相互独立）**：每个分支之间不重叠。如果一个事实可以同时归入两个分支，说明分类不够清晰。
- **Collectively Exhaustive（完全穷尽）**：所有分支合在一起覆盖了问题的全貌。如果有重要方面没被任何分支覆盖，说明有遗漏。

**常用拆解角度（根据问题性质选择）：**

| 问题类型 | 推荐拆解角度 | 示例 |
|---------|------------|------|
| 利润下滑 | 收入端 vs 成本端 | 收入=价格×销量，成本=固定+可变 |
| 增长战略 | 现有市场深耕 vs 新市场拓展 vs 新产品开发 | Ansoff 矩阵变体 |
| 竞争分析 | 行业吸引力 vs 竞争地位 vs 自身能力 | Porter+内部分析 |
| 运营优化 | 流程 vs 人员 vs 技术 vs 组织 | 按运营要素拆 |
| 进入新市场 | 市场规模 vs 竞争格局 vs 进入壁垒 vs 自身适配度 | 从外到内 |
| 问题诊断 | 现状(What) vs 原因(Why) vs 方案(How) | 通用诊断框架 |

**自检清单：**
- 把每两个 L2 议题放在一起看——它们之间有没有重叠区域？
- 想一个与核心问题相关的事实——它能被归入某一个分支吗？如果不能，说明有遗漏
- 每个 L3 子问题是否可以独立回答？如果回答它需要先回答另一个子问题，考虑合并或重新组织
- 层级是否清晰？L2 是 L1 的直接分解，L3 是 L2 的直接分解，不跨层

**Issue Tree 可视化要点：**
- 从左到右（或从上到下）展开，核心问题在最左/最上
- 同层分支数量建议 3-5 个，太少说明拆解不够细，太多说明层级不够
- 命名要具体——"市场因素"太模糊，"目标市场规模与增速"更好

## 假设驱动方法

对应 Step 3。假设驱动的思维方式是麦肯锡方法论的核心。

**为什么先假设再验证？**

新手倾向于"先收集所有数据，再看能得出什么结论"——这在时间和资源有限的情况下效率极低。假设驱动的做法是先基于经验和逻辑形成观点，然后**只收集能验证或推翻这个观点的数据**。这让分析更加聚焦。

**好假设的标准：**
- **可证伪**：能被数据推翻，而不是怎么样都对。  
  ❌ "市场竞争激烈"（永远是对的，没有信息量）  
  ✅ "前5名玩家合计市占率超过 70%，市场已进入集中化阶段"
- **有方向性**：指向一个具体的判断，而不是模棱两可  
  ❌ "成本可能是个问题"  
  ✅ "人力成本占比超过 40%，是利润率低于行业均值的主要原因"
- **可操作**：验证/推翻后能指导决策

**验证方法设计：**

每个假设都需要明确：
1. **需要什么数据**：具体的指标或信息
2. **从哪获取**：行业报告、财报、公开数据库、搜索引擎
3. **怎么分析**：定量对比、趋势分析、基准对标
4. **判断标准**：什么阈值/条件下认为假设被验证或推翻

**常见数据来源对照：**

| 数据类型 | 来源 | 适用场景 |
|---------|------|---------|
| 市场规模/增速 | IDC, Gartner, 艾瑞, Statista | 行业分析 |
| 公司财务数据 | 年报, 财报, Wind, Bloomberg | 竞争对手分析 |
| 宏观经济数据 | 国家统计局, 世界银行 | 市场环境 |
| 消费者行为 | 艾瑞, QuestMobile, 尼尔森 | 需求侧分析 |
| 技术趋势 | Gartner Hype Cycle, 专利数据库 | 技术评估 |

## 数据分析要点

对应 Step 4。数据本身没有价值，insight 才有。

**"So What?" 测试：**

每个数据发现都必须通过这个测试。如果你说不出"所以呢？这对业务意味着什么？"，这个发现就不值得放进最终输出。

- 数据："A 公司市占率 15%"  
- So What?："A 公司在行业前 5 名之外，面临被边缘化的风险"  
- So What?（更进一步）："需要在 2 年内将市占率提升至 20% 以上，否则可能失去渠道议价能力"

**分析质量检查：**
- 数据来源是否可靠？标注出处和时间
- 数据可比性：不同来源的口径是否一致？时间范围是否对齐？
- 样本偏差：数据是否能代表整体情况？
- 相关 vs 因果：数据显示的关联是否真的有因果关系？

**量化表达原则：**
- 用绝对值+相对值组合表达（"营收 50 亿，同比增长 23%"）
- 用对比增强说服力（"高于行业平均 8 个百分点"）
- 标注数据时效性（"截至 2024Q3"）
- 对估算数据标注不确定性

## 金字塔原理与故事线

对应 Step 5。好的 PPT 不是数据的堆砌，而是一个有说服力的故事。

**金字塔原理核心：**
- **结论先行**：每页的标题就是该页的核心观点，不是描述性标签
- **上下对应**：标题（观点）和正文（论据）逻辑一致
- **归类分组**：相关论点放在一起，组内逻辑清晰
- **逻辑递进**：页面之间的顺序有逻辑关系，不是随意排列

**标题写法示范：**

| ❌ 描述性标题 | ✅ 观点性标题 |
|-------------|-------------|
| 市场分析 | 目标市场规模将在 3 年内翻倍，窗口期有限 |
| 竞争格局 | 行业集中度快速上升，中小玩家加速出局 |
| 财务表现 | 利润率连续 4 季度下滑，成本端压力是主因 |
| 建议 | 聚焦高端产品线+收缩低利润业务，预计提升毛利 5pp |

**常见故事线结构：**

1. **诊断型**：问题 → 根因分析 → 解决方案 → 实施计划
2. **机会型**：市场机会 → 我们的优势 → 建议行动 → 预期回报
3. **差距型**：目标 → 现状 → 差距分析 → 弥合方案

**Dummy Page 设计要点：**
- 封面：项目标题 + 一句话总结 + 日期
- Executive Summary：3-5 个最核心的发现和建议（整个 PPT 的压缩版）
- 主体页面：每页一个核心观点 + 数据支持
- 建议页：具体可操作的行动建议 + Next Steps

## 页面内容设计

对应 Step 6。从 Dummy Pages 到完整内容的填充。

**内容密度原则：**
- 每页 3-5 个要点，不是越多越好
- 每个要点一句话说清观点，然后用数据支持
- 图表能表达的就别用文字——但图表标题也要是观点，不是描述

**数据可视化选择：**

| 要表达的信息 | 推荐图表 |
|------------|---------|
| 大小对比 | 柱状图（横向/纵向） |
| 趋势变化 | 折线图 |
| 占比构成 | 饼图/瀑布图 |
| 相关关系 | 散点图 |
| 多维对比 | 雷达图/矩阵图 |
| 流程/结构 | 流程图/组织图 |
| 时间线 | 甘特图/时间轴 |

**设计建议：**
- 配色：专业商务色系。McKinsey 蓝（#005DAA）+ 深灰（#333333）+ 白色为主
- 布局：信息层级清晰，最重要的信息最大最醒目
- 留白：不要把页面塞满，留白本身传递信息

**常见错误：**
- 标题太宽泛，读完标题不知道这页在说什么
- 数据缺乏 context（"增长 15%"——相比什么？好还是不好？）
- 堆砌太多内容，受众抓不到重点
- 逻辑跳跃，上一页和下一页之间缺少过渡
